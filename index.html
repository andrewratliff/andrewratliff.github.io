  • How to Deploy a Middleman Project to GitHub Pages Manually

    Apr 22

    I recently decided to create a personal blog using a static site framework called Middleman. Using a couple tutorials, I found it very easy to style and crank out blog posts. I soon ran into trouble though when I tried to deploy my blog to GitHub Pages. After trying to use a couple different gems (unsuccessfully), I found this post on StackOverflow that had some answers. I have adapted the advice on StackOverflow to deploy my blog as a user website instead of a project website. Below are the steps I used to get my site up and running.

    This tutorial assumes you already have a Middleman blog created on your machine. Check out the Middleman docs on setting up a blog if you haven't already.

    Setup

    config.rb

    Add the following to your config.rb file.

    activate :relative_assets set :relative_links, true

    .gitignore

    Make sure you are ignoring the build folder by adding the following to your .gitignore file:

    /build

    .nojekyll

    Disable the use of the Jekyll preprocessor. Create a blank file name .nojekyll in the source directory. You can do this from the root of your project with the touch command:

    touch source/.nojekyll

    Prepare repository

    1. Publish project to GitHub as [GitHub username].github.io
    2. From the root of your project execute bundle exec middleman build
    3. cd into the build folder that was created by the previous command.
    4. git init .
    5. git remote add origin git@github...git

    How to deploy

    This will be your routine every time you want to update your site.

    1. At the root of the project execute: bundle exec middleman build
    2. cd build/
    3. git add -A
    4. git commit -m "thoughtful commit message"
    5. git push origin master (if it's the first time pushing your project use git push -u origin master)

  • Variables as Pointers

    Apr 21

    Variables as pointers is the idea that variables point to specific place in memory. This is a concept that I didn't quite grasp when coming out of my boot camp. The best way to fully understand this idea is to take a look at some examples.

    Example 1:

    a = 'hi there'

    b = a

    a = 'not here'

    What is b in the above code? My first instinct was that b would be the string 'not here' because the second line said that b = a. In fact, b is still pointing to the string 'hi there'.

    The above diagram brings us back to the idea that variables point to physical spaces in memory; variables are essentially labels. In the third line a = 'not here' the variable a is reassigned to a new place in memory, which is a completely different string from the string that b is pointing to. The = operator assigns (or reassigns) variables. Even if the third line read: a = 'hi there', a would point to a different string than b thanks to the = operator.

    Example 2:

    a = 'hi there'

    b = a

    a << ', Bob'

    What is b in the code above? Why is it different from the first example? What is difference between the = operator and the << operator?

    As we can see from the diagram above, the third line did not reassign a. Instead, a << ', Bob' modified the original string that both a and b are still pointing to. As a new programmer, it is important to know which operators (methods) mutate the caller. Other data structures can also be mutated by methods. Methods like map! and uniq! will mutate arrays. This is a key concept that can help troubleshoot problems that arise when first starting out.
