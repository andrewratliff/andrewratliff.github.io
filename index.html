  • Variables as Pointers

    Apr 21

    Variables as pointers is the idea that variables point to specific place in memory. This is a concept that I didn't quite grasp when coming out of my boot camp. The best way to fully understand this idea is to take a look at some examples.

    Example 1:

    a = 'hi there'

    b = a

    a = 'not here'

    What is b in the above code? My first instinct was that b would be the string 'not here' because the second line said that b = a. In fact, b is still pointing to the string 'hi there'.

    The above diagram brings us back to the idea that variables point to physical spaces in memory; variables are essentially labels. In the third line a = 'not here' the variable a is reassigned to a new place in memory, which is a completely different string from the string that b is pointing to. The = operator assigns (or reassigns) variables. Even if the third line read: a = 'hi there', a would point to a different string than b thanks to the = operator.

    Example 2:

    a = 'hi there'

    b = a

    a << ', Bob'

    What is b in the code above? Why is it different from the first example? What is difference between the = operator and the << operator?

    As we can see from the diagram above, the third line did not reassign a. Instead, a << ', Bob' modified the original string that both a and b are still pointing to. As a new programmer, it is important to know which operators (methods) mutate the caller. Other data structures can also be mutated by methods. Methods like map! and uniq! will mutate arrays. This is a key concept that can help troubleshoot problems that arise when first starting out.
