I recently decided to create a personal blog using a static site framework called Middleman. Using a couple tutorials, I found it very easy to style and crank out blog posts. I soon ran into trouble though when I tried to deploy my blog to GitHub Pages. After trying to use a couple different gems (unsuccessfully), I found this post on StackOverflow that had some answers. I have adapted the advice on StackOverflow to deploy my blog as a user website instead of a project website. Below are the steps I used to get my site up and running.

This tutorial assumes you already have a Middleman blog created on your machine. Check out the Middleman docs on setting up a blog if you haven't already.

Setup

config.rb

Add the following to your config.rb file.

activate :relative_assets set :relative_links, true

.gitignore

Make sure you are ignoring the build folder by adding the following to your .gitignore file:

/build

.nojekyll

Disable the use of the Jekyll preprocessor. Create a blank file name .nojekyll in the source directory. You can do this from the root of your project with the touch command:

touch source/.nojekyll

Prepare repository

  1. Publish project to GitHub as [GitHub username].github.io
  2. From the root of your project execute bundle exec middleman build
  3. cd into the build folder that was created by the previous command.
  4. git init .
  5. git remote add origin git@github...git

How to deploy

This will be your routine every time you want to update your site.

  1. At the root of the project execute: bundle exec middleman build
  2. cd build/
  3. git add -A
  4. git commit -m "thoughtful commit message"
  5. git push origin master (if it's the first time pushing your project use git push -u origin master)
